# 커밋 컨벤션

이 문서는 inhabas.com-front 프로젝트의 커밋 메시지 작성 규칙을 설명합니다.

> 📚 **관련 문서**: [기여 가이드](./CONTRIBUTING.md) | [Merge 가이드](./COMMITER-INSTRUCTION.md)

---

## 커밋 메시지 형식

```
<type>: <제목>

<본문 (선택)>

<footer (선택)>
```

### 예시

```
feat: 게시판 키워드 검색 기능 추가

- 검색어 입력 후 엔터 키로도 검색 가능하게 수정
- 검색 결과 없을 시 안내 문구 표시

resolves: #561
```

---

## Prefix 목록

프로젝트 히스토리를 기반으로 정리한 prefix입니다.

| prefix | 설명 | 예시 |
|--------|------|------|
| `feat` | 새로운 기능 추가 | `feat: 첨부파일 다운로드 기능 추가` |
| `fix` | 버그 수정 | `fix: 회계내역 UI 깨지는 현상 해결` |
| `design` | UI · 스타일 변경 (기능 변경 없음) | `design: 버튼 간격 및 색상 조정` |
| `refactor` | 코드 구조 개선 (기능 변경 없음) | `refactor: useFetch 제네릭 타입 적용` |
| `docs` | 문서 추가 · 수정 | `docs: 라우팅 구조 문서 업데이트` |
| `chore` | 빌드 설정, 패키지, 환경 변경 | `chore: CRA → Vite 마이그레이션` |
| `style` | 포매팅, 세미콜론 등 코드 스타일만 변경 | `style: Prettier 포매팅 일괄 적용` |
| `test` | 테스트 추가 · 수정 | `test: BoardList 렌더링 테스트 추가` |
| `ci` | GitHub Actions · CI/CD 설정 변경 | `ci: 배포 워크플로우 경로 수정` |
| `revert` | 이전 커밋 되돌리기 | `revert: "feat: 검색 기능 추가" 롤백` |

> **`comment` prefix는 더 이상 사용하지 않습니다.**  
> 주석 추가/제거는 `chore` 또는 `refactor`로 작성합니다.

## 작성 규칙

### 제목

- 한국어로 작성합니다.
- 50자 이내로 간결하게 작성합니다.
- 마침표를 붙이지 않습니다.
- 명령형 어투를 사용합니다. (`추가`, `수정`, `제거`, `개선`)

```
✅ feat: 게시판 검색 기능 추가
✅ fix: 로그인 후 리다이렉트 오류 수정
❌ feat: 게시판 검색 기능을 추가했습니다.
❌ Feat: 게시판 검색 기능 추가      (대문자 금지)
❌ feature: 게시판 검색 기능 추가   (축약형 사용)
```

### 본문 (선택)

- 제목 아래 빈 줄 하나를 띄운 뒤 작성합니다.
- 변경 내용의 **무엇을**, **왜** 변경했는지 설명합니다.
- 각 항목은 `-`로 시작합니다.

```
fix: 예산지원신청 첨부파일 오류 수정

- DragNDrop 컴포넌트에서 이미지 외 파일 타입 처리 누락 수정
- BankSupportCreate에서 파일 배열 초기화 누락 수정
```

### Footer (선택)

관련 이슈가 있는 경우 작성합니다.

```
resolves: #561          # 이슈 자동 닫기
see also: #559, #557    # 관련 이슈 참고
```

## WIP 커밋

작업 중 임시 저장이 필요한 경우 WIP 커밋을 사용할 수 있습니다.  
단, **PR을 올리기 전에 반드시 `git rebase -i`로 정리합니다.**

```
# 작업 중에는 허용
WIP: 회원 테이블 UI 작업 중

# PR 전에 정리 (squash)
feat: 회원 관리 테이블 UI 추가
```

> 자세한 방법은 [Merge 가이드](./COMMITER-INSTRUCTION.md#wip-커밋-처리)를 참고하세요.

## 이전 컨벤션과의 차이

초기 프로젝트에서는 아래 형식을 사용했습니다.

```
[feat/#67] feat: 대댓글 UI 추가
```

현재는 브랜치명·이슈 번호 prefix를 제거하고 더 간결한 형식을 사용합니다.

```
feat: 대댓글 UI 추가
```

이슈 번호는 footer의 `resolves: #67`로 연결합니다.
